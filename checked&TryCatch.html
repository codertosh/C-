Checked Arrays & Try-Catch in C#

1. checked Keyword

Purpose: Enables overflow checking for integer operations

  • Default: Overflow ignored (unchecked)
  • With checked: Throws OverflowException

Array Use Cases:

  • Index calculations (a + b overflow)
  • Array size allocation validation

2. try-catch Blocks

Purpose: Handle exceptions gracefully

  • Early error detection
  • Targeted exception handling

Example Workflow

try {
    checked {
        int a = int.MaxValue;
        int b = 1;
        int index = a + b; // Throws OverflowException
        
        int[] array = new int[10];
        int value = array[index];
    }
}
catch (OverflowException ex) {
    Console.WriteLine("Overflow: " + ex.Message);
}
catch (IndexOutOfRangeException ex) {
    Console.WriteLine("Invalid index: " + ex.Message);
}

  • With checked: Immediate OverflowException

When to Use

  • Large arrays near int.MaxValue
  • Complex index calculations
  • User input-dependent sizes
  • Critical systems needing robustness
